# persistent-memory

**给兼容 AI agent 的一份、可审核的本地记忆层。**

Claude 不会自动知道你告诉 Codex 的内容，Codex 也不会自动知道你在别处保存的内容。Persistent Memory 为兼容 agent 提供可读取、可共享的上下文事实源，减少重复粘贴。

它补充平台记忆，不替代平台记忆；不会自动导入旧聊天，也不是后台服务。

## 它能做什么

- **加载基础上下文**：`加载记忆` 只读取核心文件和索引，不会读取所有项目文件。
- **经审核后保存**：AI 先提议内容和位置，用户确认后才写入。
- **按需读取细节**：只有当前话题需要时，才读取相关项目或笔记。
- **可选摘要优先加载**：活跃文件含有经审核的 `## 摘要` 时，agent 先读短摘要；只有问题需要更多细节才读取全文。没有摘要的文件保持原有行为。
- **安全管理生命周期**：归档、回收站、恢复和体检都需要确认。
- **保持透明**：记忆是可直接查看和编辑的本地 Markdown。

## 快速开始

1. 为能发现 Skills 的 agent 安装本 skill。
2. 用明确命令开始：`记住这个` 或 `remember this`。
3. 审核 AI 提议的文件和措辞，再确认。
4. 之后在兼容 agent 的新对话中说 `加载记忆` 或 `load memory`。

如果你的记忆目录建立于生命周期功能之前，请在使用归档、删除、恢复或体检前运行 `升级记忆` / `memory upgrade`。

## 工作方式

```text
~/.persistent-memory/
├── _core/                 # 身份与协作方式等基础上下文
├── _index.md              # 活跃按需记忆的一行路由
├── projects/              # 活跃项目上下文
├── notes/                 # 活跃知识与决策
└── _archive/              # 正常加载时排除
    ├── _index.md          # 归档与回收站记录
    └── _trash/            # 30 天可恢复删除缓冲区
```

`加载记忆` 加载的是**基础上下文**：`_core/` 和 `_index.md`。话题命中索引时，agent 才读取对应的活跃文件；归档文件不会被正常加载。

## 摘要优先文件

摘要优先加载对活跃按需文件是可选的。请在标题和可选引言元信息后紧接着放置 `## Summary` 或 `## 摘要`：

```markdown
# 项目标题

> 可选的引言元信息

## 摘要

- 当前状态或范围
- 关键决定或结果
- 重要限制或不确定性

## 全文细节
```

摘要使用 3–6 条事实性要点。摘要足够时，agent 只根据摘要回答；摘要不足时，agent 会明确说明并读取全文。没有有效摘要的旧文件继续按原有方式读取全文，不需要批量迁移。经审核的更新改变了已经摘要化的事实时，AI 会在同一份待确认提议中同步更新对应摘要。

## 兼容性契约

跨 agent 共享必须同时满足：

- 每个**兼容 agent**都能发现本 skill，或配置了等价的兜底指令。
- 它们使用同一台机器或同一**本地文件系统**，并将 `~/.persistent-memory/` 解析到同一位置。
- 当前用户对该位置有读写权限。

两个 agent 不会自动共享你所有历史对话；只有已经保存进该目录的记忆文件，且满足以上条件时才会共享。

如果某个 agent 不能自动发现 Skills，在它自己的配置中加入等价指令。最低兜底规则是：当用户说 `加载记忆` 时，读取 `~/.persistent-memory/_core/` 和 `~/.persistent-memory/_index.md`。

## 指令

| 你说 | 会发生什么 |
|---|---|
| `加载记忆` | 加载基础上下文：核心文件和活跃索引。 |
| `记住这个` / `save this` | 提议一项经审核的记忆更新。 |
| `更新记忆` | 将建议保存的事实与推断模式分开，等待确认。 |
| `记忆状态` | 显示活跃记忆结构和摘要。 |
| `升级记忆` | 预览旧记忆目录缺少的生命周期结构；确认后创建。 |
| `归档 <相对路径>` | 确认后将完成且低频的活跃文件移至归档。 |
| `删除 <相对路径>` | 确认后将已归档文件移至 30 天回收站。 |
| `恢复 <相对路径>` | 确认后恢复归档或回收站中的文件。 |
| `记忆体检` | 提出生命周期建议；绝不自动执行。 |

生命周期操作只接受相对路径，例如 `projects/old.md`。skill 会拒绝绝对路径、`..`、`_core/` 和控制文件。发生路径冲突时，任何文件或索引改动前都会停止；绝不自动覆盖目标文件。

## 什么该进入记忆

这里保存稳定身份、偏好、决策、精炼项目状态和原始材料的指针。

原始仓库、下载文件、媒体文件和数据集应留在原项目工作区。记忆笔记应说明它们在哪里、为什么重要，而不应把记忆目录变成通用文件仓库。

## 隐私

所有记忆都保存在本地 Markdown 文件中。请勿放入密码、API key 或其他密钥。v0.8 的生命周期与摘要规则会保护核心文件，并要求所有文件系统改动获得用户明确确认。

## 版本状态

v0.8.0 为活跃按需文件新增可选摘要优先加载；不包含 section 级加载、自动创建摘要或批量迁移。
